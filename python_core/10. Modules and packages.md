# B. Import Modules

## THEORY

Importing a file is yet another way to launch it.

### 1. Statement «import md-name»

Module — any file with extension **.py** which lays in the same folder as main script file.

`import module-name`

Don't use extension when importing modules.

When module imports it executes (only ones at loading).

Even if you import module twice per file it'll be executed once.

### 2. Statement «from md-name import obj-name»



### 2. Interactive Prompt Import

In Interactive Prompt mode module imports once at the first time (per session). 

`>>> import script1`

Make some changes in **script1** (or even can be changed the variable value)

`>>> import script1`

Won't apply changes.

### 3. Force Module Reload

Add import statement into file and for the Interactive Prompt:

`>>> from importlib import reload`

Then execute FUNCTION:

`>>> reload(script1)`

It will apply changes.

**REMEMBER! Reloads aren't transitive!**

### 4. Import part of the module

#### Use «import» statement:

`>>> import module`

Than you may access to the parts using dot:

`>>> module.part1`

kkkk

`>>> module.part1, module.part2`

('kkkk', 'nnnn')

#### Use «from» statement:

`from module import part1`

Than you may use it as **part1**

But you should remember that all code from **module** will be loaded, but for the part will be created a short name.

But if one changed some part in the module, you should use reload module to reload module and import part AGAIN.

`>>> from module import part1`

`>>> part1`

1

Change module: assign part1 var value 2.

`>>> part1`

1

`>>> from importlib import reload`

`>>> reload(module) # Only module can be reloaded`

`>>> part1`

1

`>>> from module import part1`

`>>> part`

2

#### Use «dir» function to fetch a list of all names available inside a module

`>>> dir(module)`

['__builtins__', '__cached__', '__doc__', '__file__', '__loader__', '__name__', '__package__', '__spec__', 'aaa', 'bbb', 'k']
